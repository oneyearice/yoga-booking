# 瑜伽馆课程预约系统 技术方案
**时间**：2026-03-15

## 模块划分（一句话说清楚各模块职责）
- **认证模块**：处理微信登录、手机号绑定、用户角色管理
- **课程模块**：课程CRUD、分类管理、教练关联
- **排课模块**：手动/自动排课、日历视图、冲突检测
- **预约模块**：预约/取消、候补管理、课时管理、状态跟踪
- **用户模块**：个人信息、课时余额、预约历史
- **数据统计模块**：预约统计、课程热度、课时消耗
- **API网关模块**：RESTful API、请求认证、数据验证

## 核心逻辑（人话描述，不用代码）
1. 用户通过微信登录后，根据角色权限展示不同页面
2. 管理员可以添加课程和手动/自动排课，系统自动检测时间冲突
3. 用户浏览课程和可预约时间，选择后扣除课时完成预约，约满可加入候补
4. 教练可以查看自己的课表和学员名单
5. 系统记录所有操作，提供数据统计功能
6. 后端采用Node.js + Express/Koa + MySQL/MongoDB自建服务器

## 依赖（库、API、权限）
- 微信小程序原生框架
- Node.js + Express/Koa（后端框架）
- MySQL/MongoDB（数据库）
- 微信登录授权接口
- 微信模板消息接口
- JWT（用户认证）
- 文件上传服务（课程封面图片）

## 错误处理策略（捕获哪些错误？提示语？）
- 约满：提示"预约人数已满，候补"，提供候补登记功能
- 课时不足：提示"课时不足，请充值后再预约"
- 时间冲突：提示"您已有该时段预约，支持与教练协商调剂"
- 预约取消：提前3小时免费取消，3小时内需与教练协商
- 教练冲突：排课时自动检测并提示"教练时间冲突"

## 已知风险
- 自建服务器需要考虑部署、运维、安全性
- 前后端分离需要处理跨域问题
- 小程序包大小限制，需优化资源
- 微信小程序上架需要符合审核规范

## 使用示例（预期）
- 用户打开小程序，微信一键登录
- 浏览课程列表，点击预约
- 选择时间，确认预约
- 管理员在后台管理课程和排课
- 教练查看自己的课表和学员
