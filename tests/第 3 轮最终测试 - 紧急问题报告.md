**小程序无法编译启动**

微信小程序编译器会根据 app.json 中的路径查找页面文件。当前配置下：
- 3 个页面可以正常加载
- 11 个页面会报 500 错误（文件未找到）

### 具体错误

```
Error: Page "pages/booking/confirm" not found.
Page "pages/booking/list" not found.
Page "pages/admin/courses" not found.
... (共 11 个页面)
```

## 修复方案

### 方案 A：移动文件（推荐）

将嵌套目录中的文件移动到简化路径位置：

```bash
# booking 模块
mv pages/booking/confirm/confirm.js pages/booking/confirm.js
mv pages/booking/confirm/confirm.wxml pages/booking/confirm.wxml
mv pages/booking/confirm/confirm.wxss pages/booking/confirm.wxss
rm -rf pages/booking/confirm/

mv pages/booking/list/list.js pages/booking/list.js
mv pages/booking/list/list.wxml pages/booking/list.wxml
mv pages/booking/list/list.wxss pages/booking/list.wxss
rm -rf pages/booking/list/

# admin 模块
mv pages/admin/courses/courses.js pages/admin/courses.js
mv pages/admin/schedule/schedule.js pages/admin/schedule.js
mv pages/admin/users/users.js pages/admin/users.js
mv pages/admin/dashboard/dashboard.js pages/admin/dashboard.js
# (同时移动 wxml/wxss 文件)

# coach 模块
mv pages/coach/schedule/schedule.js pages/coach/schedule.js
mv pages/coach/students/students.js pages/coach/students.js

# user 模块
mv pages/user/index/index.js pages/user/index.js
mv pages/user/bookings/bookings.js pages/user/bookings.js
```

### 方案 B：恢复 app.json 路径

将 app.json 中的路径恢复为嵌套格式：

```json
"pages": [
  "pages/index/index",
  "pages/courses/list",
  "pages/courses/detail",
  "pages/booking/confirm/confirm",
  "pages/booking/list/list",
  "pages/booking/detail",
  "pages/user/index/index",
  "pages/user/bookings/bookings",
  "pages/coach/schedule/schedule",
  "pages/coach/students/students",
  "pages/admin/courses/courses",
  "pages/admin/schedule/schedule",
  "pages/admin/users/users",
  "pages/admin/dashboard/dashboard"
]
```

同时需要修改所有跳转路径。
